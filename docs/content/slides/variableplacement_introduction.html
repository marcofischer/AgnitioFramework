Introduction to module: Variable Placement

The Variable Placement module will allow you to position elements depending on what platform is currently hosting the content.

The module will be applied if the content is running in a native app (iOS only) that is not released by Agnitio. It is intended to be used with content that will run on the Agnitio platform AND another platform where there are native elements taking up parts of the screen.

Module
Implemented in v2.4.1

Requirements

  • agnitio.js

Script reference

The module should be placed in the modules folder and then the following need to be added to index.html:

<script src=”modules/variable-placement/variable-placement.js”></script>

Constructor Syntax

app.variablePlacement.init(rules, moveMenu)

Parameters

rules array
Instructions to the module what elements to move and how.
moveMenu boolean [true]
A flag to set if the module should try to find the main menu and move it.

Rules

A rule is an object that includes instructions on how to find and move elements.

The rule will always include a query string that will be used to find the element. Additionaly, either a CSS property and its value or a class name will be included.

query
A query string to find elements. Same format as given to querySelector.
property
A valid CSS style property. Will normally be either left or margin-left.
value
The value to set to the above property. Should include the unit, e.g. "150px".
classname
A classname to assign to the element(s) found.

See the second example below to understand how these are set.

Moving Main Menu

By default the module will try to find the presentation menu and move it 150px to the left (absolute positioning assumed). This is the most common usecase for using this module and is therefor provided out of the box.

For this to work the menu should have the id mainmenu, which is the default used by Agnitio menu modules.

If another id is used for the menu, or another distance is desired then set the moveMenu parameter to false and provide a specific rule for the menu.

Examples

Example: Initializing Variable Placement without rules

The following example will try to find the menu and move it 150px from the left.

app.variablePlacement.init();

Example: Initializing Variable Placement with two rules

The following example will try to find the main menu AND two elements and move them. One by setting the style property directly on the element, and the other by adding a class to the element (the class should be defined in a CSS file).

app.variablePlacement.init([
  {query: ".platform2", property: "margin-left", value: "150px"},
  {query: "#references", classname: "left-150"}
]);

Example: Initializing Variable Placement to not look for menu

The following example is the same as the above except that it will not try to find the main menu (the addition of the second paramenter false).

app.variablePlacement.init([
  {query: ".platform2", property: "margin-left", value: "150px"},
  {query: "#references", classname: "left-150"}
], false);
