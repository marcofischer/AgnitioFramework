app.init

Summary

Loads initial content.

Core Presentation Method
Implemented in v2.0

Syntax

app.init(structure, content, subcontent)

Parameters

structure string
Name of slideshow or collection to load.
content string (optional)
Slide or section to initially be displayed.
subcontent string (optional)
Slide in collection to initially be displayed.

Description

app.init will load either the content specified by the parameters or via the search string in the URL (takes presedence).

app.init is by default called from the bottom of the setup.js file.

Loading content via URL search string

It is possible to tell the presentation what content to load by adding a search string to the URL. The functionality is built into the app.init method. There are two patterns of search string that can be used:

  • ?path=structure.content.subcontent
  • ?slide=slideName

This could for example be used when developing to easily set what slide should be displayed when reloading browser.

Examples

Example: app.init of a collection with name 'placebo'

The following example will load the collection 'placebo' if it exist, and scroll to the 'efficacy' section.

app.init('placebo', 'efficacy');
