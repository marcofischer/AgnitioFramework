Introduction to module: Data

DEPRECATED MODULE! Use analytics module instead

The data module will allow you to easily monitor slides in a presentation.

Currently the data module is very basic and simply does one thing: monitor slides in a presentation (send slide data to Agnitio Analyzer). The module will get slide names/strings from one of three ways (in order):

  1. monitormap, a global JavaScript object
  2. data-monitor attribute on slide elements
  3. slide id (fallback)

Convention

At Agnitio we use a JavaScript object to map slideshow and slide ids with descriptive strings that should be logged in the Analyzer each time a slide is entered. This makes it easy to have an overview of the data that is submitted and to update it.

See below for an example of a monitormap object.

Limitations

Current version does not allow to specify slides that should be excluded from monitoring. This will be added in future version.

Constructor Syntax

app.data = new Data(monitoringEnabled);

Parameters

monitoringEnabled boolean (optional)
If the framework should submit data to the Analyzer or not. Possible values are true or false. Defaults to true.

Examples

Example: a global monitormap object

The following is an example of a monitormap object used by the data module to get the data strings it should submit to the Analyzer.

window.monitormap = {
  presentation: "Placebo v2",
  slideshows: {
    intro: "Placebo introduction",
    efficacy: "Efficacy",
    moa: "Mode of Action"
  },
  slides: {
    IN_info: "General information",
    EF_firstline: "Efficacy First-line",
    MO_video: "Mode of Action video"
  }
};

Example: data-monitor attribute on slides

The following is an example of a data-monitor attribute on the opening article-tag of a slide (set on each slide).

<article id="efficacy" class="slide" data-monitor="10001 - Efficacy">
