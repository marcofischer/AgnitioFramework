Introduction to module: Analytics

The analytics module will allow you to easily monitor slides in a presentation.

The objective of the analytics module is very simple: monitor slides in a presentation (send slide data to Agnitio Analyzer).

Initialize module

In setup.js:

app.analytics.init(config);

Parameters

config object (optional)
See below for configuration options.

Configuration options

excludeContent array (optional)
Collections and slideshows that should be excluded from monitoring.
excludeSlides array (optional)
Slides that should be excluded from monitoring.
map object (optional)
A JavaScript object mapping structure ids to ids specified by client.
offsetChapters integer (optional)
Used, in example, to not use the root structure as a chapter.
version string (optional)
The version of the presentation being monitored.

Examples

Example: a global monitormap object

The following is an example of a monitormap object used by the analytics module to get the data strings it should submit to the Analyzer. In the map each key is the ID used in the code to reference the content (i.e. the ID used in presentation.json).

window.monitormap = {
  efficacy: {
    id: "chapter-002",
    name: "Efficacy"
  },
  placebo: {
    id: "chapter-001",
    name: "Placebo v2"
  }
  high_altitude: {
    id: "slide-014",
    name: "Efficacy at high altitude"
  }
};

Example: setup with configuration

The following is an example of how to initialize the module with some cofiguration in the setup.js file. In this example the ids and names that should be monitored has been mapped in the app.monitormap object. The offsetChapters option tell the module to ignore the first structure level for monitoring.

app.analytics.init({
  version: '1.3.1',
  map: app.monitormap,
  offsetChapters: 1
});
