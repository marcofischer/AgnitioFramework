How to use the bundled interface

The dynamic agenda module comes bundled with an example interface that can be easily plugged into any presentation using the framework. However, the interface is kept simple by design as most presentations/products will want to use their own custom interface for creating dynamic agendas.

.

Limitations

The interface does not currently have functionality for moving slides around in an existing agenda.

Initializing the interface (and dynamic agenda module)

Add the following script tags to the index.html file:

<script src="modules/dynamic-agenda/dynamic-agenda.js"></script>

<script src="modules/dynamic-agenda/dynamic-agenda.interface.js"></script>

Add the following CSS import to the all.css file:

@import url("modules/dynamic-agenda/dynamic-agenda.interface.css");

Copy the two html files in the modules/dynamic-agenda/slides folder to the slides folder.

In setup.js:

app.agenda.init(name, config);

(This replaces the general initialization instruction of dynamic agenda)

Parameters to app.agenda.init

name string
The name to be used in localStorage to store the agendas.
config object (optional)
Interface configuration. See below for options.

Configuration options

pathToThumbs string
Specify a path to the thumbnails of the slides. Defaults to content/img/thumbs
slides array
Specify what slides should be available in the dynamic agenda. Defaults to all slides in presentation.json file.

Accessing the interface

Once the interface has been initialized it is possible to load it with app.load('DA');.

If the presentation is using the AutoMenu module, you can prepend or append a link to the menu by adding the following to the menu configuration:

prepend: [
  {title: 'Builder', goTo: 'app.load("DA")'}
]

Examples

Example: Initializing the interface

The following example would create a dynamic agenda that stores created slideshow in localStorage.DA_Placebo. This call would normally be placed in setup.js but can also be placed in the code setting up the interface for the dynamic agenda.

app.agenda.init('DA_Placebo');

Example: Initializing the interface with limited slides

The following example would create a dynamic agenda that will only use the slides provided in the configuration.

app.agenda.init('DA_Placebo', {
  slides: [
      'intro',
      'risk_of_osteo',
      'mode_of_action',
      'types_of_pain',
      'treatment_adherence',
      'side_effects',
      'studies',
      'pain_therapy'
    ]
  });
