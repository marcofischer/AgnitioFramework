Getting elements in slides

The framework provides a convention for getting DOM elements in sides, which will then be automatically cleaned up when exiting the slide.

To allow the browser garbage collector to effectively release memory it is important to remove references to elements. Using the convention described below will allow the framework to automatically clean up any references to DOM elements when exiting a slide.

Slide elements

To make use of this feature an elements object is added to the JavaScript object for the slide. The object contains the name that should be used to reference the elemnts, and a query string used by the framework to find the element.

The element is then referenced through this.element.

The query string should be the same as used in a querySelector call.

If referencing a list of elements, the query string is replaced by an array with two items. The first item is the query string (same as discussed above), and the second is the string all. The framework will then use a querySelectorAll to get the elements.

Examples

Example: Adding an elements object to a slide with query strings for several elements.

(function() {
  var slide = app.slide.efficacy = {
    elements: {
      pictureArea: ".image-container",
      allText: ["#text_li li", "all"],
      indicators: [".indicator .dot", "all"],
      rotation: "#image_rotation"
    },
    onEnter: function(ele) {
      console.log('Elements available here');
      // Add class to pictureArea element
      util.addClass(slide.element.pictureArea, 'fadeIn');
      // Adding event listener to the "rotation" element
      app.addEvent('tap', slide.rotateImage, slide.rotation);
    },
    onExit: function(ele) {
      console.log('No need to clean up elements here');
    }
  }
})();
